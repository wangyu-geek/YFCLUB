# 迁移文件格式

当前版本的数据迁移入口支持 JSON 文件。推荐由旧版 `club.mdb` 导出工具生成如下结构：

```json
{
  "sourceVersion": "legacy-mdb-export-1",
  "members": [
    {
      "legacyPk": "member-1",
      "memberNo": 1001,
      "name": "张三",
      "gender": "男",
      "birthMonth": "05",
      "birthDay": "12",
      "mobile": "13800000000",
      "pointsBalance": 120,
      "totalSpent": 860.5,
      "lastConsumeAt": "2026-04-01 09:20:00",
      "remark": "旧系统导入"
    }
  ],
  "consumptions": [
    {
      "legacyPk": "consume-1",
      "memberLegacyPk": "member-1",
      "amount": 35,
      "pointsAdded": 3,
      "operatorName": "系统导入",
      "remark": "历史消费",
      "createdAt": "2026-03-18 15:30:00"
    }
  ],
  "gifts": [
    {
      "legacyPk": "gift-1",
      "giftName": "水杯",
      "pointsCost": 30,
      "stockQty": 20,
      "status": "ACTIVE",
      "remark": "常规礼品"
    }
  ],
  "redemptions": [
    {
      "legacyPk": "redeem-1",
      "memberLegacyPk": "member-1",
      "giftLegacyPk": "gift-1",
      "giftName": "水杯",
      "qty": 1,
      "pointsUsed": 30,
      "operatorName": "系统导入",
      "remark": "历史兑换",
      "createdAt": "2026-03-20 16:00:00"
    }
  ],
  "settings": {
    "storeName": "永丰文体",
    "pointsRuleAmount": 10,
    "legacyJpdj": "10",
    "defaultOperator": "管理员"
  }
}
```

说明：

- `members` 是必需主数据，`legacyPk` 必填，用于导入幂等。
- 如果 `consumptions` 和 `redemptions` 都为空，系统会按“余额导入模式”处理。
- 如果包含历史消费或兑换，系统会按“全历史导入模式”处理，并在导入结束后做积分余额校正。
- 同一 `legacyPk` 或同一批次文件重复导入时，系统会通过批次指纹和映射表阻止重复入库。
